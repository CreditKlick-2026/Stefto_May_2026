# Backend Architecture Plan for Stefto Frontend

## Project Overview
- **Frontend**: React 19 + Vite + Tailwind CSS
- **Backend Type**: Scalable RESTful API with microservices-ready architecture
- **Database**: MongoDB with Redis caching layer
- **Media Storage**: Cloudinary for all blog images and uploads

---

## Tech Stack Recommendation

| Layer | Technology |
|-------|------------|
| Runtime | Node.js 20+ with TypeScript |
| Framework | Express.js or Fastify |
| Database | MongoDB with Mongoose ODM |
| Caching | Redis (ioredis client) |
| Image Storage | Cloudinary |
| Authentication | JWT + Refresh Tokens |
| API Documentation | Swagger/OpenAPI |
| Rate Limiting | express-rate-limit |
| Validation | Zod |
| Logging | Winston + Morgan |

## API Endpoints Structure

### 1. Authentication Module (`/api/auth`)
```
POST   /api/auth/register          - User registration
POST   /api/auth/login             - User login (returns JWT)
POST   /api/auth/refresh           - Refresh access token
POST   /api/auth/logout            - Invalidate tokens
POST   /api/auth/forgot-password   - Password reset request
POST   /api/auth/reset-password    - Password reset confirmation
GET    /api/auth/me                - Get current user profile
PUT    /api/auth/profile           - Update user profile
```

### 2. Blog Module (`/api/blogs`)
```
GET    /api/blogs                  - List blogs (paginated, cached)
GET    /api/blogs/:slug            - Get single blog post
POST   /api/blogs                  - Create blog (admin)
PUT    /api/blogs/:id              - Update blog (admin)
DELETE /api/blogs/:id              - Delete blog (admin)
GET    /api/blogs/search           - Search blogs
GET    /api/blogs/categories       - List categories
GET    /api/blogs/tags             - List all tags
```

### 3. Contact Module (`/api/contact`)
```
POST   /api/contact                - Submit contact form
GET    /api/contact                - List submissions (admin)
GET    /api/contact/:id            - Get single submission (admin)
PUT    /api/contact/:id            - Update submission status (admin)
DELETE /api/contact/:id            - Delete submission (admin)
```

### 4. Newsletter Module (`/api/newsletter`)
```
POST   /api/newsletter/subscribe   - Subscribe to newsletter
POST   /api/newsletter/unsubscribe - Unsubscribe
GET    /api/newsletter/subscribers - List subscribers (admin)
```

### 5. Upload Module (`/api/upload`)
```
POST   /api/upload/image           - Upload image to Cloudinary
DELETE /api/upload/image/:publicId - Delete image from Cloudinary
```

## Redis Caching Strategy

### Cache Keys Pattern
```
blogs:list:{page}:{limit}:{category}:{sort}     - Blog list
blogs:slug:{slug}                               - Single blog by slug
blogs:categories                                - All categories
blogs:search:{query}:{page}:{limit}            - Search results
contact:list:{page}:{limit}                     - Contact submissions
newsletter:count                                - Subscriber count
```

### Cache Invalidation Strategy

#### On Blog Create/Update/Delete:
1. Delete `blogs:list:*` patterns (all paginated lists)
2. Delete `blogs:categories`
3. Delete specific blog by slug: `blogs:slug:{slug}`
4. Delete search caches if applicable
5. Use Redis UNLINK for async deletion

#### On Contact Update:
1. Invalidate contact list cache for admin users

### Cache TTL Settings
| Data Type | TTL | Reason |
|-----------|-----|--------|
| Blog list | 15 min | Frequently updated |
| Single blog | 30 min | Rarely changes |
| Categories | 1 hour | Rarely changes |
| Search results | 5 min | High variability |

### Cache-Aside Pattern Implementation
```
1. Check Redis for data
2. If cache miss → query MongoDB
3. Store result in Redis with TTL
4. Return data
```

## Pagination Implementation

### Cursor-Based Pagination (Recommended for Performance)
```typescript
// Request
GET /api/blogs?page=1&limit=10&sort=-publishedAt

// Response
{
  success: true,
  data: [...blogs],
  pagination: {
    currentPage: 1,
    totalPages: 5,
    totalItems: 47,
    itemsPerPage: 10,
    hasNextPage: true,
    hasPrevPage: false,

  }

```

### Query Parameters
- `page` - Page number (default: 1)
- `limit` - Items per page (default: 10, max: 50)
- `sort` - Sort field with prefix (e.g., `-publishedAt` for descending)
- `category` - Filter by category
- `search` - Search query

## Cloudinary Integration

### Image Upload Flow
```typescript
// 1. Client requests signed upload URL
// 2. Server generates signed URL with preset
// 3. Client uploads directly to Cloudinary
// 4. Cloudinary returns public_id and URL
// 5. Server saves to MongoDB
```

### Upload Preset Configuration
- Folder: `stefto/blogs` or `stefto/users`
- Max file size: 5MB
- Allowed formats: jpg, png, webp
- Transformations: auto-optimize, auto-format

## Authentication Flow

### JWT Token Strategy
```
Access Token: 15 min expiry, stored in memory (not localStorage)
Refresh Token: 7 days expiry, httpOnly cookie
```

### Token Refresh Flow
```
1. Access token expires
2. Client receives 401
3. Client calls /api/auth/refresh with refresh token
4. Server validates refresh token
5. Returns new access token + new refresh token
6. Client retries original request
```

### Security Headers
```
Authorization: Bearer {access_token}
```

## Frontend Integration Guide

### API Client Setup (Axios)
```typescript
// api/client.ts
import axios from 'axios';

const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL || '/api',
  headers: { 'Content-Type': 'application/json' }
});

api.interceptors.request.use((config) => {
  const token = localStorage.getItem('accessToken');
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

api.interceptors.response.use(
  (response) => response,
  async (error) => {
    if (error.response?.status === 401) {
      const { data } = await axios.post('/api/auth/refresh', {}, { withCredentials: true });
      localStorage.setItem('accessToken', data.accessToken);
      error.config.headers.Authorization = `Bearer ${data.accessToken}`;
      return axios(error.config);
    }
    return Promise.reject(error);
  }
);
```

## Notes for Developer

1. **Start Simple**: Begin with Express + MongoDB, add Redis later
2. **Cache First**: Implement caching early to avoid refactoring
3. **Admin Panel**: Consider building an admin panel for managing blogs/contacts
4. **Security**: Never expose secrets in frontend code
5. **Testing**: Add unit tests for services, integration tests for APIs
